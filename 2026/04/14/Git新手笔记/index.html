Git 基础认知

核心概念

  • 工作区（Working Directory）：本地正在编辑的文件目录，肉眼可见、可直接修改。

  • 暂存区（Staging Area）：临时存放待提交的修改。

  • 本地仓库（Local Repository）：存放在本地电脑里的版本库，记录所有提交记录，可回溯、回滚。

  • 远程仓库（Remote Repository）：存放在服务器（如GitHub、GitLab）的仓库，用于团队协作、代码备份。

  • 分支（Branch）：独立的开发线路，可在不影响主分支的前提下开发新功能，完成后合并。

  • 储藏（Stash）：临时保存未提交的修改，用于切换分支、处理冲突时，暂存当前工作内容，不丢失代码。

注意事项

(Github) Commit规范

  • 一般来说commit文本具有较为通用的规范。
  • 如果项目使用Git-cliff这种自动生成changelog的插件，最好遵守命名规范确保commit被正确分类。
常见commit分类 对应内容
fix bug修复
feat 新功能
docs 文档
perf 性能优化
refactor 代码重构
style 样式
test 测试
chore 日常维护
ci 持续集成
security 安全
revert 撤销

其他

  1. Git主要用于对文本文件进行版本控制。对二进制文件（Excel、图片、PDF等）的支持非常有限，无法实现增量更新。不要在VSCode中处理Excel、图片等二进制文件的冲突。

  2. 执行 git reset --hard，会丢失未保存的修改（Stash内容不受影响）。执行git push --force可能误删远程代码。

  3. 可在 .gitignore 中配置忽略项。以免提交无用文件（如日志、缓存、虚拟环境）
